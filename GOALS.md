# Ghostline 目标全景（完整且实用版）

> README 是愿景，ROADMAP 是做的顺序，这份是**做到什么程度才算「完整且实用」**。
> 判断标准只有一条：**几个朋友愿意把它当成日常工具长期用下去，而不是新鲜两天就删。**

---

## 「完整且实用」到底意味着什么

一个加密聊天 app 之所以被普通人抛弃，几乎从来不是因为「加密不够强」，而是因为这些:

1. **换手机 = 聊天记录全没了 / 账号进不去** —— E2EE 应用的头号劝退点。
2. **消息偶尔丢 / 顺序乱 / 转圈不发出去** —— 不可靠比没功能更致命。
3. **每一步都要猜「这按钮干嘛的」** —— 没有空状态、没有加载提示、报错看不懂。
4. **找不到上次那句话** —— 没有搜索、没有联系人管理。

所以「完整且实用」= **能用的功能 + 可靠 + 可恢复 + 体面的 UX**，四者缺一不可。
下面把目标按这四块全部铺开，并标好优先级。

**优先级定义**
- **P0 必须有**:没有它，朋友用两天就会删。
- **P1 应该有**:决定「能用」和「好用」的差距。
- **P2 锦上添花**:有最好,砍了不影响日常。

## 一、账号与密钥(E2EE 应用的生死线)

| 目标 | 优先级 | 说明 |
| --- | --- | --- |
| 邀请码注册 | P0 | 已在计划内。 |
| **密钥恢复 / 备份** | **P0** | **最重要的新增目标。** 换手机/重装后能找回身份和读历史消息。方案见下。 |
| 安全码 / 公钥指纹核对 | P1 | 双方线下核对一串指纹,确认中间没人冒充——E2EE 信任的最后一环。 |
| 多设备 | P2 | 真正的多设备同步很难,先做「一人一设备」,以后再说。 |
| 改名 / 头像 / 个人简介 | P1 | 不然通讯录里全是一串 ID,认不出谁是谁。 |

### 密钥恢复方案(P0,起步就要想清楚)
E2EE 的私钥只在本地——丢了就什么都解不开。给朋友圈应用的务实方案,二选一或都做:
- **助记词 / 恢复短语**:注册时生成一串恢复词,用户自己抄下来。换设备时输入它在本地重建私钥。
  (私钥可由恢复词确定性派生,服务器永远拿不到。)
- **加密云备份**:用一个由「恢复短语派生的密钥」加密整份私钥+历史,存服务器。
  服务器只见密文,用户凭恢复短语解开。
> 起步阶段至少实现「助记词」这一条,否则切片 3 一上加密,第一个换手机的朋友就流失了。

## 五、可靠性(比功能更重要,却最容易被忽略)

> 这一整块是「玩具」和「能日常用」的真正分水岭。每条都是 P0/P1。

| 目标 | 优先级 | 说明 |
| --- | --- | --- |
| **断线自动重连 + 消息补偿** | **P0** | 网络抖动后能自动重连,并补回断线期间漏掉的消息。 |
| **发送队列 + 失败重试** | **P0** | 发送中/发送失败要有明确状态,弱网下不丢、可重发。 |
| 消息去重 + 顺序保证 | P0 | 每条消息带客户端生成的唯一 ID + 序号,防重复、防乱序。 |
| 本地优先(乐观更新) | P1 | 点发送立刻上屏(标记发送中),不等服务器确认,体感才跟手。 |
| 启动离线可读 | P1 | 没网也能看历史消息(本地已解密缓存)。 |
| 推送可靠送达 | P1 | 杀进程后仍收得到「新消息/收到信」。 |

## 明确的「不做」清单(守住朋友圈应用的边界)

把这些写下来,是为了**防止把项目做成第二个 Signal 而永远发布不了**:

- ❌ **语音 / 视频通话**:技术栈(WebRTC + 媒体服务器)完全是另一个项目,砍掉。
- ❌ **真正的群组加密协议(MLS / sender keys)**:逐人扇出已够小群用。
- ❌ **多设备实时同步**:一人一设备 + 恢复短语已覆盖换机场景。
- ❌ **联邦 / 去中心化 / 自托管分发**:一个朋友共用的服务器就够。
- ❌ **公开注册 / 用户发现 / 陌生人社交**:邀请制是初心,不碰增长。
- ❌ **机器人 / 小程序 / 支付 / 朋友圈动态**:不做平台,只做通信。

> 每当想加新功能,先问:**「我的几个朋友会因为没有它而删 app 吗?」** 不会,就进 P2 或不做。

## 「完整且实用」验收清单(达成即算 v1.0)

当下面每一条都为真,这个 app 就从「学习项目」变成了「真能用的产品」:

- [ ] 一个朋友能凭邀请码注册,抄下恢复短语,加你为好友,发出第一条加密消息。
- [ ] 他换了台手机,用恢复短语登回来,**还能读到以前的消息**。
- [ ] 弱网下发消息有明确的「发送中/失败/重发」,且**最终不丢、不重、不乱序**。
- [ ] 群里能引用回复、能 @、能丢个表情回应。
- [ ] 他能写一封信、设「明天送达」,你第二天**收到推送并拆开**。
- [ ] 他能搜到上周那句话、能把吵闹的群静音、能给 app 加把锁。
- [ ] 杀掉进程,新消息推送照样弹出来。
- [ ] 任何一处加载/出错,屏幕上都有**人话提示**,而不是白屏转圈。

> 达成这 8 条 = 完整且实用的 v1.0。其余 P2 都是 v1.0 之后的事。
